    Cohort / M窗口 / Take Rate
    • Cohort 按首贷放款月份划分；首贷显式使用 user_order.secondary_loan = 0，首贷用户数与首贷订单数当前校验一致。
    • 分析维度默认展示整体；credit_level 来自首贷订单最新一条 risk_model_predict.output_data 的 $.credit_level；该字段仅从 2026-04-01 后开始作为可解释分层，2026-04-01 前只保留整体口径。
    • M1/M2/M3 按首贷放款后 30 天滚动窗口统计：M1 为 0~29 天，M2 为 30~59 天。
    • 仅完整观察窗参与对比；未成熟窗口显示为 --，不计入结论。
    • 首贷 Take Rate 仅纳入最后一期应还日 +15 天 <= 观察截止日的首贷订单，未完整表现订单不进入分子和分母。
    • 合同金额使用 fin_application.principal；实际放款金额与权责表一致，优先按 fin_application.pay_request_id = payment_record.request_id 且 payment_record.status = 1 取 payment_record.amount，缺失时用 principal - due_fee1 兜底。
    • Take Rate 使用合同金额 base：净现金 Take Rate = (实收服务费 + 实还本金 + 实收利息 + 实收罚息 - 合同金额) / 合同金额。
    • 实收字段与权责表保持一致，直接取 repayment_schedule 当前 paid_fee1、paid_principal、paid_interest、paid_penalty，不再用 repayment_time 过滤。
    • 如果 LTV 与权责表观察截止日不同，通常是两张报表刷新时间不同导致；跨报表校验时需要统一 observe_date 后再比较最近月份。
    • 页面中的息费率 = 实收服务费、利息、罚息 / 合同金额；本金未回收率 = 未回收本金 / 合同金额；净现金 Take Rate = 息费率 - 本金未回收率。
    • 复贷 Take Rate 当前取 M4 累计窗口口径，仅纳入最后一期应还日 +15 天 <= 观察截止日的复贷订单；M4 未成熟时用历史成熟 M4 均值预测。
    • 当前仍是资产现金流口径，暂未纳入资金成本、CAC、催收成本和运营费用。
